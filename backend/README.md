# Playwright Automation Backend

Enterprise Express backend for the Playwright Automation Dashboard.
This service controls test execution, serves artifacts, streams real-time progress via REST + Socket.IO, and maintains execution history.

## Architecture

- **Express.js**: REST API for execution management, artifact retrieval, and settings.
- **Socket.IO**: Real-time event streaming for live execution progress, logs, and artifacts.
- **Playwright Runner**: Spawns and manages `child_process` instances of the Playwright framework, parsing `stdout`/`stderr` to report structured progress.
- **MongoDB Persistence**: Stores both live and historical executions, settings, and artifact metadata securely using Mongoose.
- **In-Memory Cache**: Tracks live process handles and limits heavy database roundtrips during execution streaming.
- **Artifact Scanner**: Automatically detects and registers reports, screenshots, videos, and traces generated by Playwright, making them accessible via static file serving and persistent via the Artifact model.

## Setup & Running

1. **Install dependencies:**
   ```bash
   npm install
   ```
2. **Configure environment:**
   Copy `.env.example` to `.env` and adjust the values. The most important variable is `PLAYWRIGHT_PROJECT_PATH`, which should point to the root of the Playwright testing framework.
3. **Start the server (Development):**
   ```bash
   npm run dev
   ```
4. **Start the server (Production):**
   ```bash
   npm run start:prod
   ```

## REST API Endpoints

- `GET /health` - Server health status
- `POST /api/executions` - Start a new test execution
- `GET /api/executions` - List all executions (live + history)
- `GET /api/executions/:id` - Get details of a specific execution
- `POST /api/executions/:id/stop` - Abort a running execution
- `DELETE /api/executions/:id` - Delete an execution and its artifacts
- `GET /api/reports` - List available HTML reports
- `GET /api/screenshots` - List available screenshots
- `GET /api/videos` - List available videos
- `GET /api/traces` - List available trace files
- `GET /api/settings` - Retrieve global settings (environments, suites, browsers)
- `PUT /api/settings` - Update global settings

## Real-Time Events (Socket.IO)

Clients should connect to the root namespace (`/`) and listen for:

- `execution-started`
- `execution-progress`
- `execution-log`
- `execution-completed`
- `execution-error`
- `execution-report`
- `execution-screenshot`
- `execution-video`
- `execution-trace`
